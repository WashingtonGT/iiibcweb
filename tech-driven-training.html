The literacy spine

Everyone starts on Module B

Before any role track, every person who touches an AI tool takes the AI Literacy Baseline. It is the EU AI Act Article 4 literacy floor — logged as proof — and the prerequisite for every other module. Four disciplines run through the rest of the kit:

Position

Set a deliberate AI posture by domain — aggressive adopter, fast follower, or risk-conservative — that leadership owns and the board can govern.

Build

Turn proprietary data into a moat: grounded RAG, spec-first agentic engineering, and an LLMOps layer that survives model updates you don't control.

Secure

Treat AI output as a hostile draft until proven otherwise — secure coding, OWASP LLM red-teaming, and sanctioned tools that beat shadow AI.

Measure

Govern, comply, and prove value — risk-tiered systems, auditable operating model, and ROI with token cost in the denominator.

The curriculum

Thirteen modules, mapped to your readiness assessment

Each module is a self-contained kit: a business case, measurable objectives, a timed agenda, core content, hands-on labs with answer keys, takeaway templates, a knowledge check, a facilitator guide, and an outcome scorecard. Start everyone on Module B, then prioritize the modules whose assessment sections scored lowest. All thirteen modules are live.

How to run the kit

Assess, deliver, measure

1 · Assess

Run the tech-driven readiness assessment for a section-by-section profile. Prioritize the modules whose sections scored Foundational or Developing.

2 · Deliver

Start everyone on Module B (logged literacy floor), then deliver the mapped modules along each role track and sequence with the 90-day rollout.

3 · Measure

Each module ends with an outcome scorecard. Track value with the three-tier ROI model — adoption, depth/efficiency, and financial impact with token cost in the denominator.
